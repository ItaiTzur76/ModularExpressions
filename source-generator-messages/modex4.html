ModularExpressions

Source-Generator Error MODEX4

Property 'property' in class 'class' is not an arrow property so it cannot be used as part of the calculation of a GenerateModex attribute

A Modex property must be an expression-bodied read-only property in order to participate in the calculation of a modular expression.
This calculation (done at compile-time) is the only purpose of such properties - they have no functionality at runtime - so declaring them with getters or setters would be meaningless, anyway.

Example

The following sample generates MODEX4:

// MODEX4.cs
using static ModularExpressions.Modex;

namespace MyNamespace;

public partial class MyClass
{
    [GenerateModex(nameof(StartingWithNonWordCharacterModex))]
    public static partial string StartingWithNonWordCharacterPattern();   // MODEX4
    //                           ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

    private static Modex StartingWithNonWordCharacterModex { get { return Beginning + NonWordCharacter; } }
}
